Premières explications avant les commandes:

Master est la branch principale, c'est celle de notre projet.
Les branch sont créées seulement quand on a peur de faire des collisions avec les fichiers déjà existants.
En effet, si l'on travaille sur des choses totalement différentes, il vaut mieux ne pas créer de branch, c'est inutile.
Quand en revanche, ce que l'on fait risque de foutre le bordel dans le projet, il vaut mieux créer une branch.
Pour rassembler notre branch avec master on fait un merge

Avant de commencer des modifications, toujours faire un git pull

Quelques commandes:

Etapes pour mettre à jour le projet (sur la branche courante) :

git add *
git commit -m "message de ce qui a été modifié"
git push

Commandes branch

aller sur une branch : git checkout nom de a branch
regarder le statut de la branch (les modifications): git status
créer une branch: branch nom de la branch
supprimer une branch: git branch -d nom de la branch Faites attention à ce que vous supprimez !!

Etapes pour merger:

git checkout master
git merge nom de la branch
git push
